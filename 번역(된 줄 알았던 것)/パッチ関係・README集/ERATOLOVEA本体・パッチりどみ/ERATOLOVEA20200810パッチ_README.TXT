【참조 파일】
	EVENT_P.ERB EVENT_S.ERB

@PREGNANCY_CHECK(임신 했는지 판정한다)(와)과@IN_VAGINA_ALL(임신상대가 누군가를 판정한다)가 불리는 것이
아침이 되었을 때의 1일 1회만(@EVENT_NEWDAY내)을 위해서(때문에),
다음과 같은 경우에 상정외의 캐릭터의 아이를임신 하는 경우가 있습니다

례) 아침의 조교로 노예 A를 주인 B로 조교해,질 쏘아 맞히고
	밤의 조교로 노예 C를 주인 D로 조교(질 쏘아 맞히고 없음)
	(전제로서 조교시에 조수는 없는 노예 A·C 모두 노역을 실시하지 않았다)
	이튿날 아침이 되어임신체크가 실행되고 노예 A의임신가 확정했을 경우,
	A는 D의 아이를임신 하고 있다

임신체크 및임신상대가 누군가를 판정하는 함수는,그 노예가 1일의 질 비쳐진 수를 이용하고 있습니다
그 노예가 1일의 질 비쳐진 수는,노예의 플래그(CFLAG:101,103-105)에 질 비친 상대 마다 격납됩니다
그러나,주인이나 조수인가 노예나 그 이외(노역시의 손님 따위)의 4개로 분류되어 기록된다
한편,1일 안에 주인이나 조수가 변경되어도 리셋트 되지 않기 때문에
상기와 같은 일이 일어난다고 생각됩니다
	A는 주인에게임신 당한→이튿날 아침의 시점에서의 주인은 D→A는 D에임신 당했다
	밤에 질 비친 상대에임신 당했을 경우는 문제 없고 상대가 판정될 것입니다

단순하게임신체크 및임신상대가 누군가를 판정하는 회수를 1일 2회로 하는 대처로 좋은 것인지
판단하기 어려워 하고 있으므로 고치고 있지 않습니다
	1일 2회로 하면 꽤임신 하기 어려워지는 것 같은 생각이 듭니다
	체감인 것으로 실제 어떤가,또 때를 봐 확인합니다

●처녀·동정·백버진를 빼앗은 상대의 표시가
실제로 빼앗은 상대와 다른 경우가 있다
----------------------------------------------------
※사양인 것이나 버그인 것이나 판별이 되지 않기 때문에 고치고 있지 않습니다

처녀·동정·백버진를 빼앗은 상대는
빼앗긴 캐릭터의 CSTR:10(처녀) CSTR:11(동정) CSTR:12(백버진)에 기록되고 있습니다
(TRACHECK.ERB 에 해당 처리 부분)

그러나,이하와 같은 상황이라면 실제로 빼앗은 상대와 다른 상대가 기록되는 일이 있는 것 같습니다
예) 조교에 대해,조교자가 조수에 교대하고 있는 상황으로
조교자(조수)가 삽입 커맨드에 의해 노예의처녀·동정·백버진를 빼앗았을 경우
기록되는 이름이 마스터의 이름이 된다

마스터가 조수에 명령해 노예의처녀·동정·백버진를 빼앗게 했다,라고 생각할 수 없는 것도 아니기 때문에
사양인 것인가,버그인 것인가의 판별이 되지 않고 고치고 있지 않습니다

※거치고 싶고 실마리응의 안면 플래그는,상기의 예에서는 노예의처녀·동정·백버진를 빼앗은 상대로서
조수를 기록하고 있다

※마스터가 조교자인 채,조수삽입명령커맨드에 의해 노예의처녀·동정·백버진를 빼앗았을 경우
기록되는 이름은 조수의 이름
